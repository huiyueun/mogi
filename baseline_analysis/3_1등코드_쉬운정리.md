# ③ 1등 코드 (GOH30) — 전문가 3명이 각자 그리고 평균낸다

> 파일: `[Private_LB 1st] 코드 공유.ipynb`
> 성적: Private LB R-Hit@1cm **0.7035** (1등)
> 이름 뜻: **G**RU + **O**DE + **H**yperPhysics, 합쳐서 **30**개 모델

---

## 한 줄 요약

> **"미리 만든 후보 중에서 고르지 않는다. 생각하는 방식이 서로 다른 전문가 3명이 각자 답을 그리고, 그 답들을 평균낸다."**

② 후보 고르기와 가장 큰 차이가 바로 이거다.

| | ② 후보 고르기 | ③ 1등 코드 |
|---|---|---|
| 방식 | 후보 28개 중 **고르기** | 후보 없이 **직접 예측** |
| 점수 천장 | 71.9%에 묶임 | 후보 천장에 안 묶임 |
| 성적 | 68.2% | **70.35%** |

후보가 없으니 "후보 천장"에 갇히지 않는다. 그게 더 높은 점수의 핵심 비결이다.

---

## 전문가 3명 (각자 생각하는 방식이 다름)

비슷한 모델끼리 모으면 **같은 실수를 같이** 한다. 반대로 **다르게 생각하는 모델끼리 평균내면 실수가 서로 상쇄**된다. 그래서 일부러 성격이 다른 3종을 섞었다.

1. **GRU — 순수 시계열 전문가**
   과거 점들의 흐름을 양방향으로 읽고(앞→뒤, 뒤→앞), "다음은 여기쯤"을 감으로 잡는다. 중요한 시점에 더 집중하는 attention도 붙어 있다.

2. **Neural ODE — 물리 적분 전문가**
   "속도와 가속도가 이러니까, 0.08초를 시간으로 차근차근 적분하면 여기"라고 **미분방정식을 풀어서** 위치를 구한다. (공기 저항 같은 감속 항도 학습으로 넣음.)

3. **HyperPhysics — 선회 물리 전문가**
   모기가 **비행기처럼 몸을 기울여 도는(뱅킹 턴)** 물리를 공식으로 직접 박아넣었다. 직진 구간에선 회전을 안 켜고, 실제로 꺾는 순간(선회각·속력이 임계값을 넘을 때)만 회전을 발동한다.

이 3명을 각각 **시드만 바꿔 10번씩** 학습 → 총 **30개 모델의 답을 등가중 평균**한다.

---

## 3명이 공통으로 쓰는 "비법 양념" 5가지 (여기가 점수의 핵심)

세 모델 모두 아래를 똑같이 쓴다.

1. **잔차만 학습 (가장 중요)**
   0부터 위치를 맞히는 게 아니라, 먼저 **등속 외삽으로 대충 찍어두고(57.9%짜리 base)**, 거기서 빗나간 차이(잔차)만 학습한다. 문제가 훨씬 쉬워지고 안정적이다. → ①번 등속 외삽이 여기서 토대로 쓰인다.

2. **방향 정렬(yaw)**
   모기가 향한 방향을 항상 +x축으로 돌려놓고 본다. 모기가 어디로 날든 같은 모양으로 보이니 학습이 쉬워진다. (②번의 Frenet 좌표계와 같은 발상.)

3. **Soft R-Hit 손실 (아주 중요)**
   "평균 오차를 줄여라"가 아니라 **"1cm 안에 들어가게 해라"는 평가지표 자체를 직접 학습 목표**로 삼는다. 1cm 적중을 미분 가능한 형태로 근사해서 모델이 직접 최적화한다. 0.7대 진입의 공통 기법.

4. **내부 전이 사전학습**
   궤적 하나에서 중간 지점들(예: 5·6·7·8번째까지만 보고 그다음 맞히기)도 정답처럼 써서, **데이터 1만 개를 5만 개로** 불려 연습시킨다.

5. **EMA + Y-flip TTA**
   학습 가중치를 부드럽게 평균내고(EMA), 예측할 때 입력을 좌우 대칭으로 한 번 더 예측해 평균낸다(TTA). 둘 다 점수를 안정적으로 굳히는 장치.

---

## 전체 흐름

```
[과거 11개 점]
      │
      ▼  (등속 외삽으로 base를 깔고, 방향 정렬)
      │
      ├─→ GRU 10개      ┐
      ├─→ Neural ODE 10개 ├─ 각자 "잔차"를 예측
      └─→ HyperPhysics 10개 ┘
                          │
                          ▼  (Y-flip TTA로 각자 한 번 더 예측해 평균)
                          │
                  30개 답 등가중 평균
                          │
                          ▼
                  [최종 예측] → submission_GOH30.csv
```

---

## 실행 관련 메모

- 데이터는 `Data/train`, `Data/test`, `train_labels.csv`만 제자리에 두면 된다.
- `FROM_SCRATCH=True`면 30모델을 처음부터 학습(맥북 MPS 기준 약 2시간 10분), 끝나면 `models_goh30/`에 저장된다.
- 한 번 학습한 뒤 `FROM_SCRATCH=False`로 두면 저장된 가중치를 불러와 약 2분 만에 재현된다.
- 외부 파일 없이 **이 노트북 하나로 raw 데이터 → 전처리 → 학습 → 제출까지** 전부 끝난다.

## 세 코드를 관통하는 교훈

| | 방식 | 성적 | 핵심 |
|---|---|---|---|
| ① 등속 외삽 | 가던 대로 간다 | 57.9% | 모든 코드의 토대(base) |
| ② 후보 고르기 | 물리 후보 28개 중 선택 | 68.2% | 안전하지만 후보 천장(71.9%)에 갇힘 |
| ③ 1등 코드 | 다른 전문가 3명이 직접 예측 후 평균 | **70.35%** | 천장을 버리고 직접 예측 + 지표 직접 최적화 + 앙상블 |

**핵심 변화:** ② → ③ 으로 가면서 "**후보 고르기를 버리고, 평가지표(1cm)를 직접 학습 목표로 삼아, 서로 다른 모델을 앙상블**"한 것이 점수를 끌어올린 결정타다.
